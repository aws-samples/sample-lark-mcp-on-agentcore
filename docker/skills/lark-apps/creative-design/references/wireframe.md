# 线框图

帮助用户快速探索设计想法。提问遵循 `lark_get_skill(domain="apps", section="creative-design/creative-design")`「提问」一节：关键信息缺失且承重时先做一轮聚焦提问，否则基于合理假设直接铺方案。生成多个粗略的线框图，在锁定方向之前把设计空间勾勒出来。优先追求广度而非精细打磨：默认每个想法给出 2-3 种明显不同的方案，用户明确要求广度探索时再加。用简单的形状、占位文字和极少的颜色，把焦点留在结构和流程上。整体保持手绘草图的感觉——手写风格但清晰可读的字体；以黑白为主、点缀少量颜色；低保真、简洁。多方案默认铺进一个画布式布局（见 `lark_get_skill(domain="apps", section="creative-design/creative-design")`「如何开展设计工作」）；本 MCP 环境不提供画布脚手架文件，请直接手写画布外壳，用内联 SVG/CSS 兜底。单个画板内部的局部变体用页内开关承载（本 MCP 环境不提供 Tweaks 控件面板脚手架，请直接手写对应控件）。
